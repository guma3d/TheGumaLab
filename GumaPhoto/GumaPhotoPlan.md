# Guma Photo Plan (가족 특화 시맨틱 검색 시스템)

## 최종 목표
- 가족 구성원의 얼굴을 학습시키고 모든 사진에서 인물 정보를 추출하여 벡터 DB에 저장
- 인물의 표정(기분), 위치 정보, 이미지에서 보이는 날씨 및 계절 등 다양한 컨텍스트 정보를 벡터화 및 DB화
- 홈서버에 웹 서비스를 구축하고 로컬 LLM과 연동하여 자유로운 자연어 검색 ("비 오는 날 웃고 있는 OO이 사진 찾아줘" 등)을 통해 가족 사진을 쉽게 찾을 수 있도록 시스템 구현

---

## 1단계: 개발 환경 구성 및 홈서버 인프라 구축
가족 사진 검색 웹 서비스를 구동할 홈서버의 기초 인프라를 Docker 기반으로 세팅합니다.

1. **Docker Compose 구성**: 강력한 백엔드 시스템 구동을 위해 벡터 데이터베이스(예: Qdrant, Milvus)와 로컬 LLM 분석 인프라를 `docker-compose.yml`로 통합합니다.
2. **웹 서비스 스택 준비**: FastAPI(백엔드API)와 모던 웹 프레임워크(예: Next.js 또는 React/Vue 기반)를 이용해 사용자가 접근할 웹 애플리케이션의 초기 구조를 잡습니다.
3. **학습 및 저장 구조 세팅**: 가족 구성원의 기준 얼굴(Reference Faces)을 모아둘 디렉토리와 방대한 사진들을 순차적으로 읽어들일 처리 큐(Queue) 시스템을 설계합니다.

## 2단계: 맞춤형 안면 인식 및 정보 추출(멀티모달) 파이프라인 개발
사진 한 장에서 얻을 수 있는 최대한의 의미 정보를 추출합니다.

1. **가족 구성원 특화 얼굴 학습 모듈**: 
   - `FaceNet`이나 `DeepFace` 등의 모델을 사전에 세팅한 가족 구성원 사진에 파인튜닝(또는 임베딩 거리 비교)하여 높은 정확도로 인물을 식별해 내는 로직을 구현합니다.
2. **복합 메타데이터 및 맥락 분석 모듈**: 
   - **인물 및 표정**: 식별된 각 가족 구성원의 감정(행복, 놀람, 피곤 등) 추출.
   - **환경 및 컨텍스트**: CLIP 등의 이미지 캡셔닝 모델을 사용하여 이미지의 전반적 맥락(실내/실외, 날씨, 계절, 주변 사물 등)을 텍스트로 치환.
   - **EXIF 데이터 결합**: 사진 파일 자체의 시간(Date) 및 위치 정보(GPS/위도/경도)를 결합하여 정확한 시공간적 정보를 매핑합니다.

## 3단계: 임베딩 변환 및 벡터 데이터베이스 적재
추출된 방대한 텍스트 메타데이터와 원본 이미지를 고차원 벡터로 변환하여 검색 시스템의 기반을 다집니다.

1. **텍스트-이미지 통합 벡터화**: 식별된 인물 정보, 장소, 시간, 기분 등의 종합 메타데이터 문장을 LLM 임베딩 모델을 통해 벡터로 변환합니다.
2. **백그라운드 배치 프로세서 구축**: 가족 사진 수만 장을 홈서버가 무리하지 않는 선(예: 야간 시간대 백그라운드 작업, 발열/리소스 모니터링 연동)에서 연속적으로 DB에 밀어 넣는 데몬(Daemon) 스크립트를 개발합니다.

## 4단계: LLM 검색 연동 및 웹 애플리케이션 개발
사용자의 모호한 질문도 찰떡같이 알아듣는 자연어 검색 로직과 UI를 연결합니다.

1. **AI 기반 검색 중개자(LLM Agent) 구현**: 
   - 사용자가 "작년 여름 바닷가에서 신난 아빠 사진"이라고 검색하면, LLM이 이를 분석하여 벡터 DB 쿼리 파라미터(`Person: 아빠`, `Emotion: 행복`, `Season: 여름`, `Place: 바다`)로 똑똑하게 분리·변환하도록 프롬프트를 설계합니다.
2. **홈서버 웹서비스 프론트엔드 제작**: 
   - 검색창 하나만으로 작동하는 구글 라이크의 심플하고 직관적인 웹 인터페이스 개발.
   - 검색된 이미지들의 갤러리 뷰 제공 및 상세 메타데이터 표시 화면을 구축합니다.

## 5단계: 배포 및 글로벌 가이드라인 동기화
TheGumaLab 가이드라인에 따른 홈서버 구동 및 테스트

1. **GitHub 경유 배포**: 로컬 PC에서 개발 및 테스트가 완료된 소스를 GitHub 리포지토리에 푸시하고 홈서버로 풀(Pull) 받아 배포합니다.
2. **환경변수(.env) 보안 동기화**: `scp` 등의 방식을 통해 API 키 및 DB 패스워드를 홈서버에 수동 전송합니다.
3. **가족 대상 실서버 테스트**: 스마트폰이나 태블릿 등을 통해 홈서버 웹페이지에 접속하여 언제 어디서든 빠르고 정확하게 가족 사진 검색 기능이 구동되는지 최적화합니다.
