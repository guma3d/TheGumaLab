# GumaPhoto 프로젝트 마스터 플랜 (가족 특화 시맨틱 검색 시스템)

이 문서는 GumaPhoto 서비스의 전체 기획과 향후 로드맵을 정의합니다. (수시로 업데이트 됩니다)

## 📌 최종 목표
- 가족 구성원의 얼굴을 학습시키고 모든 사진에서 인물, 표정, 배경(장소, 분위기) 정보를 다각도로 추출하여 벡터 DB에 영구 보존.
- 모바일(스마트폰)에서 무제한 대량의 사진을 즉각적으로 업로드하고, 백그라운드에서 실시간으로 갤러리를 정렬/분석하는 무중단 동기화 서비스 구축.
- 사용자의 자연어(예: "가족들이랑 해변가에서 웃고있는 사진", "아빠가 군복 입은 사진")를 찰떡같이 알아듣는 LLM 기반 사진 검색 엔진 개발.
- 홈서버(FastAPI+Nginx) 위에서 가족 누구나 웹/모바일로 쉽게 접근 가능한 구글 포토(클라우드 갤러리) 수준의 개인화 갤러리 런칭.

---

## ✅ 1단계: 개발 환경 구성 및 홈서버 인프라 구축 (완료)
1. **Docker 기반 환경 세팅**: `docker-compose.yml`을 통해 메인 FastAPI 애플리케이션(`gumaphoto_app`) 및 Qdrant 벡터 DB(`gumaphoto_qdrant`) 구축.
2. **홈서버 연동 지원**: `TheGumaLab` 메인 포털 인덱스에 GumaPhoto 추가 및 Nginx 리버스 프록시(`/GumaPhoto`) 매핑 완료.

## ✅ 2단계: 자동 스마트 분류 파이프라인(Organizer) (완료)
1. **시계열 & 메타데이터 구조화**: 기존 뒤섞인 원본 사진들을 `YYYY/YYYY-MM/YYYY-MM-DD_01.jpg` 형식의 규격화된 트리 구조로 자동 재배치. (`organizer_pipeline.py`)
2. **Junk(찌꺼기) 자동 필터링**: 쓸모 없는 스크린샷, 심하게 흔들린 사진, 완전 중복 사진 강제 삭제 및 용량 최적화 수행.
3. **폴더 컨텍스트 영구 보존**: 과거에 의미 있게 묶어둔 폴더명(예: `2005_성욱_군대`)을 DB에 `original_context`라는 이름으로 보존하여 추후 벡터 정보의 그라운드-트루스(Ground Truth)로 활용.

## ✅ 3단계: 멀티모달(얼굴+배경) 하이브리드 벡터화엔진 (완료)
1. **안면 인식 (InsightFace)**: 최대 수준의 안면 추출 엔진(`buffalo_l`)을 가동하여 사진에 등장한 인물들의 512차원 벡터와 성별/나이 추출.
2. **표정 분석 (DeepFace)**: 추출된 얼굴 영역만 따로 분석하여 'happy', 'sad', 'surprise' 등 감정 상태 부여.
3. **분위기 & 배경 분석 (최신 CLIP)**: `clip-ViT-B-32` 오리지널 이미지 인코더를 가동하여 사진의 전반적인 분위기와 피사체 정보를 벡터(512차원)로 합축.
4. **Qdrant 멀티-벡터 아키텍처**: "scene(배경)", "face(얼굴)" 네임드 벡터를 분리 생성하여, 하나의 사진(Point) 안에 모든 시맨틱 정보를 하이브리드로 주입시키는 대량 배치 스캐너(`vector_indexer.py`) 구축 및 8,500장 가동 성공.

## 🚀 4단계: FastAPI 백엔드 검색 엔진 및 UI/UX 고도화 (예정)
1. **LLM 기반 마법 검색 엔진**: 사용자가 자연어로 입력한 검색어를 비전-언어 모델(또는 로컬 LLM)이 `{"대상": ["아빠", "엄마"], "분위기": "해변", "감정": "happy"}` 형태로 똑똑하게 파싱하여 Qdrant 검색 조건(Filter & Vector Match)으로 자동 치환.
2. **구글 포토(Google Photos) 스타일 타임라인 갤러리 뷰**: 검색을 하지 않고 홈 화면에 처음 들어왔을 때, 전체 사진을 연도별/월별로 매우 부드럽게 스크롤하며 감상할 수 있는 기본 타임라인 UI 설계.
3. **편리한 선택 및 앨범 저장 UI**: 검색된 사진 결과물 중 마음에 드는 사진들을 다중 선택(체크박스)하여 '내 스마트폰으로 다운로드' 하거나 '임시 앨범'에 묶어 저장할 수 있는 사용자 친화적 편의 기능 추가.

## 🚀 5단계: 모바일 실시간 업로드 및 동기화 (UploadToGumaPhoto) (예정)
1. **무제한 대량 업로드 지원**: 스마트폰에서 수백장의 사진을 서버로 전송할 수 있는 강력한 스트리밍 전송 포트 설계.
2. **동기화 파이프라인 (Worker)**:
   * 파일 수신 ➡️ `organizer_pipeline.py` 호출 (분류 및 자동 저장) ➡️ `vector_indexer.py` 호출 (실시간 Qdrant 벡터 적재)
3. **실시간 진행상태(Progress) UI**: 서버 창을 닫아도 업로드/처리 현황을 이어볼 수 있는 중단 없는 Stateful 프로그레스 바 제공.

## 🤖 6단계: 사용자 참여형 AI 피드백 및 자가 학습 시스템 (Active Learning) (예정)
1. **미식별 인물(Unknown Face) 수집**: 사진 스캔 시 기존 학습된 인물(가족 4인) 외에 얼굴이 선명하게 나온 사람이 있다면, 임시 ID를 부여하고 '미식별 인물 DB'에 따로 큐잉(Queue).
2. **온디맨드(On-Demand) 피드백 버튼 및 대화형 UI**: 평소에는 묻지 않다가, 사용자가 메뉴에서 능동적으로 '피드백 하기' 버튼을 눌렀을 때만 미식별 사진을 1장씩 보여줌.
3. **LLM 기반 자연어 피드백 분석**: AI가 "여기에 파란 자켓을 입은 인물의 정보가 없으니 누구인지 알려주면 내가 학습하겠습니다."라고 물으면, 사용자가 "아, 이 분은 할아버지야." 라고 자연스럽게 대답. 이 자연어 텍스트를 LLM(언어 모델)이 분석하여 어떤 인물인지 맥락을 파악하고 라벨링(Tagging)을 자동으로 처리.
4. **자동 데이터 최신화 (AI 자가 진화)**: 사용자가 태그를 부여하면 해당 인물 벡터를 새로운 기준점(Reference vector)으로 기억함. 이후 배치 스케줄러가 과거의 'Unknown' 상태 사진들을 스스로 재-벡터화 스캔하여 새롭게 인지한 인물(할아버지)을 찾아내 정보 업데이트.

## 🛠️ 유지보수: 파일-DB 자가 치유 아키텍처 (예정)
1. **사진 직접 삭제 동기화 (레이지 클린업)**: 홈서버(C드라이브) 탐색기에서 오빠가 불필요한 사진을 직접 삭제했을 경우, 갤러리 검색 렌더링 직전에 `os.path.exists()`를 체크하여 사라진 경우 화면 표출 우회 및 DB 자가 삭제 (유령 데이터 방지).
2. **주기적 데이터 정합성 배치 (CronJob)**: 매일 새벽 3시에 서버 내 실제 보유 파일 리스트와 Qdrant 내부 데이터를 대조(Diff)하여 유령 데이터 대청소(Garbage Collection)를 자동으로 가동하는 안전장치.
