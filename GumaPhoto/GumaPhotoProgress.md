## 📌 [2단계] 맞춤형 안면 인식 파이프라인 개발 진행 중

### 테스트 1: 초기 안면 인식 모델 (`DeepFace`) 도입 검증
1.  **테스트 내용:** 인물 다수(가족)가 등장하는 샘플 해변 사진을 생성하여 `test_ai.py` 스크립트를 통해 평가
2.  **결과:** 
    *   인물 감지(Detection) 및 보편적 속성(나이, 감정 등) 식별에는 일부 성과가 있었으나, 어린이의 나이/성별을 성인으로 오인하는 등 Demographic 추론 정확도 개선이 필요함을 확인.
3.  **개선 조치:**
    *   더 높은 정확도의 백엔드(Retinaface 검출기)를 도입하여 복수 인물 분리 정확도를 끌어올렸으나 로컬 환경에서의 한계 극복을 위해 엔진 교체 결정.

### 테스트 2: 고성능 얼굴 추출 모델 (`InsightFace`) 업그레이드
1.  **테스트 내용:** 오픈소스 최고 수준의 안면 인식 라이브러리인 InsightFace (`buffalo_l` 모델) 도입
    *   Dockerfile에 C++ 컴파일 도구(`g++`, `build-essential`)를 추가하고 `onnxruntime-gpu` 최신 환경 구성 완료.
2.  **결과 (대성공):**
    *   샘플 해변 사진에서 3명의 얼굴을 놓치지 않고 완벽하게 Crop(추출)해냄.
    *   특히 여성, 남성 등 성별 식별력과 복수 객체의 크기 차이 분석 면에서 압도적으로 우수한 성능을 증명함.
    *   벡터 DB에 저장하기 위한 핵심 요소인 **512차원 얼굴 임베딩(Vector)**을 성공적으로 생성함을 확인함.

### 테스트 3: 모델 앙상블 (`InsightFace` 추출 + `DeepFace` 감정 분석) 구축
1.  **테스트 내용:** `InsightFace`의 정확한 위치 추적/추출 능력과 `DeepFace`의 뛰어난 표정/감정 판독 능력을 하나로 합치는 앙상블 파이프라인 구성.
2.  **프로세스:**
    *   [InsightFace] 사진 전체 스캔 (640px 최적화 리사이징을 통해 해상도 무관 초고속 스캔 보장 가능성 확인) 
    *   👉 얼굴 영역 추출 및 512차원 고유 벡터값 획득
    *   👉 [DeepFace] 추출된 네모 얼굴 조각 이미지만 넘겨받아 'happy', 'fear' 등의 표정 상태 텍스트화
3.  **결과:** 한 번의 스크립트 실행으로 한 사진 내 각 다중 객체의 **[성별, 예측 나이, 512차원 아이디, 감정 상태]** 등 가족 검색 서비스를 위한 완벽한 메타데이터 기반 뼈대 세팅 완료.

## 📌 [3단계] AI 기반 갤러리 정리 자동화 파이프라인 (완벽 적용 완료 🎯)
*   **물리적 폴더 시계열 구조화 자동화 및 파일명 정규화**
    *   인물 기준의 비효율적인 물리 구조(용량 낭비)를 배제하고 오직 시간/위치 기반의 트리 뼈대 유지 
    *   (`YYYY/YYYY-MM/YYYY-MM-DD_이벤트/`) 형태로 사진 EXIF/GPS 기반 자동 이동 
    *   **파일명 일괄 정규화 (Renaming):** 
        *   뒤죽박죽인 원본 파일명(`KakaoTalk_...`, `IMG_1234.jpg`)을 버리고, 해당 폴더 내에서 **`YYYY-MM-DD_01.jpg`, `YYYY-MM-DD_02.jpg`** 등 시간순서(초 단위 혹은 임의)의 일련번호를 부여하여 완벽하게 깔끔한 이름으로 재지정함. (동영상은 확장자 유지 반영)
    *   **예외 처리 (메타데이터 누락 및 복구 프로세스):**
        *   **시간 정보 누락:** 
            *   파일 이동/복사 중 왜곡될 확률이 높으므로 **파일 속성(생성/수정일자)은 절대 참고하지 않음.**
            *   대신 해당 파일이 위치했던 **기존 상위 폴더 2단계의 이름**을 체크하여 연도/월 등의 시간 텍스트를 유추.
            *   단서가 전혀 없다면 `Unknown_Date` (연도 미상) 폴더로 분류 및 격리.
            *   👉 *추후 오빠가 피드백을 주면, Unknown 폴더들만 대상으로 재필터링 배치 파이프라인 가동.*
        *   **위치/장소 누락:** 
            *   날짜는 있지만 장소가 없는 경우 `YYYY-MM-DD_Unknown_Location` 으로 폴더 생성.
            *   👉 *추후 오빠가 피드백을 주면, Unknown Location 폴더들을 대상으로 재필터링 통과.*
        *   **시간 & 위치 모두 누락:** 
            *   완전 미상인 `Unknown` 폴더로 격리 보관 후, 동일하게 추후 피드백을 통해 재필터링 수행.
*   **사전 찌꺼기(Junk) 및 완전 중복 사진 1차 필터링 (Pre-processing)**
    1.  **완전 중복 파일 제거 (Hash 필터):** 파일의 용량과 해시값(MD5/SHA256)을 비교하여 이름만 다르고 완전히 똑같은 사본(Copy) 파일들을 0순위로 즉시 삭제 (스토리지 낭비 방지).
    2.  **쓸모없는 사진 즉각 삭제(과감한 필터):** 
        *   주차장 기둥 번호 캡처본 등 "위치 기록용(임시)"으로 찍은 사진
        *   택배 송장 원본, 영수증, 단순 문서 촬영본, 스마트폰 긴비율 스크린샷 
        *   **초점이 완전히 나간 심령사진(Extreme Blur):** 피사체를 도저히 알아볼 수 없는 극도로 흔들린 사진
        *   👉 이처럼 가족 앨범과 전혀 무관하거나 복구 불가능한 찌꺼기(Junk) 사진들은 임시 격리조차 하지 않고 **자비 없이 영구 삭제 (B컷 보관함 제외)** 처리하여 서버 용량을 최적화함.
    3.  **동영상(Video) 전용 처리 규칙:** 
        *   `.mp4`, `.mov` 등의 동영상 파일도 **완전 중복 해시(Hash)** 검사는 동일하게 수행하여 중복 본은 삭제함.
        *   화면 녹화본(Screen Recording) 등은 스마트폰 해상도 비율이나 파일명 규칙으로 찌꺼기(Junk) 처리.
        *   눈감음/흔들림 같은 AI 이미지 분석은 동영상 생태계와 맞지 않으므로 패스(Bypass)하고, 날짜/위치 메타데이터(MP4 atoms 등)를 읽어 정규 트리 폴더(`YYYY/YYYY-MM/`)로 바로 직행(이동) 조치.
*   **유사 사진(수동 연속 촬영) 베스트 컷 추출 알고리즘**
    1.  **시공간 기반 그룹핑**: 연사(Burst) 기능 외에도, 비슷한 시간대(예: 1~2분 이내)에 촬영되고 배경/구도(이미지 해시/유사도)가 겹치는 사진들을 하나의 '유사 촬영 그룹'으로 묶음.
    2.  **흔들림 탈락**: OpenCV 라플라시안 분산을 이용한 흔들림(blur) 검증
    3.  **눈감음 탈락 (핵심)**: `InsightFace` 106 랜드마크 튜닝을 통한 눈꺼풀 거리 비(EAR) 측정으로 눈 감은 사진 폐기
    4.  **베스트컷 선정**: `DeepFace` 감정 분석을 돌려 웃는 표정(happy)에 최고 가산점 부여. 만약 다수가 정상이라면 구도/화질 점수를 추가 비교.
    5.  **보관함 이관**: 베스트 1장만 원본 폴더에 유지하고, 나머지 B컷들은 영구 삭제 전 오빠가 확인할 수 있도록 `[임시보관함_유사사진]` 등으로 이관.
*   **대규모 파이프라인 안전장치 (Fail-Safe & Resume)**
    *   **DB 기반 체크포인트 (이어하기):** 수만 장 처리 도중 서버가 재부팅되어도, 내부 SQLite 등의 가벼운 DB에 "IMG_001 처림 완료"를 기록해 두어 처음부터 다시 도는 대참사 방지.
    *   **원본 문맥 영구 보존:** `2005_성욱_군대` 등 기존에 정리해둔 원본의 귀중한 폴더 컨텍스트를 SQLite에 별도 칼럼(`original_context`)으로 저장하여 다음 단계(멀티모달 AI 스캔)의 핵심 땔감으로 활용. 조치 완료.
    *   **원본 무결성 보존:** 1차 정리 시에는 원본을 강제로 `이동(Move)`하거나 `삭제(Delete)`하지 않고, **`복사(Copy)` 또는 `하드링크(Hard Link)` 방식**으로 새로운 폴더 트리에 배치. 오빠가 최종 확인 후 "완벽하다!" 승인하면 구 폴더를 날리는 투트랙 안전 방식 적용.

## 📌 [4단계] 멀티모달 AI(CLIP, YOLO) 기반 심층 메타데이터 추출 및 벡터화 (완벽 적용 완료 🎯)
*   **목표:** 단순한 EXIF를 넘어서, 픽셀 텍스처를 AI 혼종 파이프라인(YOLO-X + CLIP-L + InsightFace)이 함께 씹어먹고 의미 단위로 추출하여 텍스트 및 다차원 벡터/페이로드 인덱스 DB(Qdrant) 구축.
*   **완료된 파이프라인 (업그레이드 완료):** 
    *   `YOLO-X`: 컨피던스를 0.15로 대폭 낮추고 imgsz를 1024로 튜닝하여, 백그라운드 구석의 먼지 같은 `cup`, `dog` 사물까지 극한으로 다 긁어냄.
    *   `CLIP-L-14`: 한국어 다국어 제한 모델을 치우고, 가장 최신형/초고도 영어 텍스트 인코더인 `ViT-L-14` (768차원)으로 업그레이드 완료. 프롬프트를 영어 전용으로 개조하여 인식률 대폭 향상 성공.
    *   `No People` 강제 태깅: 미등록 객체나 사람이 0명인 사진을 명확히 필터링할 수 있도록 Fallback 기능 탑재.
*   **백엔드 구조 및 OOM 방어 적용:** 멀티모달 스레드 중첩으로 인한 서버 크러시와 VGPU 메모리 부족 현상 증상을 완벽하게 방어 (TF Growth 활성화, Thread Lock, GC 도입).

## 📌 [5단계] 검색 엔진 및 컴팩트 갤러리 UI/UX 고도화 (완벽 적용 완료 🎯)
*   **LLM 기반 검색 (Gemini)**: 사용자의 복잡한 자연어 검색어("송이 비오는날")를 분석하여 `people`, `location`, `scene`으로 정확히 분할 파싱해주는 Gemini-3.1-lite 통합 API 연동.
*   **극강의 메타데이터 UI**: 모바일과 PC 모두를 고려하여 날짜, 시간대, 계절, 장소, 사람 태그를 0.5rem 초소형 폰트로 압축 배치. 정보 누락 시 회색(`Unknown`) 뱃지를 띄워 사용자 피드백을 유도하는 능동형 디자인 적용 완료.

## 📌 [6단계] 모바일 실시간 연동 및 지능형 업로드 파이프라인 (완벽 적용 완료 🎯)
*   **목표:** 휴대폰이나 웹에서 사진을 대량으로 업로드하면, 즉각적으로 GumaPhoto 앱 화면에서 백그라운드 파이프라인이 자동 연속 실행되어 폴더 분류와 DB 인덱싱을 끝마침.
*   **핵심 적용 완료 기능:**
    1.  **FastAPI Background Tasks 통합:** 파일 업로드를 받자마자 응답(200 OK)을 프론트엔드로 바로 반환하여 끊김 없는 UX 제공 후, 백엔드 워커에서 `organizer_pipeline.py` -> `vector_indexer.py` 순으로 연쇄 스케치 가동.
    2.  **스토리지 정리 (JUNK & Move):** 프론트에서 받은 파일들을 임시 폴더에서 연도-월 정규 폴더로 `shutil.move` 완전히 이동 처리 및 영구 삭제 적용.

## 📌 [7단계] 사용자 참여형 AI 피드백 및 딥러닝 자가 학습 (On-Demand Active Learning) (완벽 적용 완료 🎯)
*   **단일 사진 로컬 재학습 (InsightFace + Gemini 통합)**: 모달 창에서 제공되는 `피드백 폼`을 통해 "이 사진은 성욱이야" 라고 남기면, 백엔드가 Gemini를 호출해 정답(Target Person) 추출.
*   해당 사진에서 가장 큰 얼굴 임베딩(512D Vector)을 즉석에서 새로 뽑아 `known_faces.pkl` 파일에 배열 형태로 즉시 추가(Append) 학습. 
*   동시에 해당 사진의 Qdrant Vector Payload 정보를 실시간 Patch(수정)하여 서버 재기동 없이 즉각적인 검색 품질 향상.

## 📌 [유지보수 및 아키텍처] 3단계 앱 내 완전 삭제 프로세스 연동 (완벽 적용 완료 🎯)
*   **목표:** 모달 창의 "휴지통" 아이콘을 눌렀을 때 사진 파일을 삭제할 뿐만 아니라 관련된 모든 DB 데이터까지 완벽하게 함께 소거하는 3중 Hard Delete 전략 구현.
    *   **1단계. 물리적 스토리지 삭제:** 홈서버(`app/data/organized/...`) 안의 원본 이미지 리소스 강제 삭제(`os.remove`).
    *   **2단계. Qdrant 벡터 증발:** 벡터 공간상에 남아있는 특정 사진의 상황/분위기 점 포인트 제거.
    *   **3단계. SQLite 비석 처리:** 스케줄러가 삭제된 파일을 또 인덱싱하려 드는 좀비 부활 사태를 막기 위해 해당 파일 레코드를 `DELETED` 로 덮어쓰기 완료.

## 🚀 [8단계] 사용자 UX 개선 및 대규모 AI 자가 학습(Self-Learning) 파이프라인 (다음 목표)
*   **객체 태그(Objects) 상세 UI 표시:** 검색 결과 목록의 썸네일에서는 가림 현상을 방지하기 위해 사물 태그를 숨기고, 사진을 클릭하여 나타나는 상세 모달(큰 화면) 내부에서만 YOLO가 뽑은 사물 태그들을 보여주도록 프론트엔드 UI 수정.
*   **업로드 진행 상태 UX 적용:** 여러 장의 사진을 올릴 때 프론트엔드에서 서버의 백그라운드 인덱싱 상황을 실시간(예: Processing 10/30)으로 인지할 수 있도록 업로드 진행 상황바 또는 뱃지 피드백 구현.
*   **자연어 기반 메타데이터(장소/시간/인물) 일괄 수정 (자연어 피드백 업데이트):** 
    *   사용자가 단일 사진의 피드백 창에 "이거 2021년 여름에 강남에서 찍은 성욱이 사진이야" 등의 자연어를 입력하면, 백엔드의 Gemini LLM이 이를 완벽히 파싱하여 `Date`, `Location`, `People` 필드 3가지를 한꺼번에 추출.
    *   단순 모달용 1회성 피드백에 그치지 않고, 파싱된 내용을 바탕으로 원본 JSON이나 Qdrant Payload의 장소/시간 속성을 실제로 교체.
*   **1주일 단위 통합 DB 재학습 스케줄링 (자가 학습 파이프라인):**
    *   일주일 단위로 쌓인 '사용자 피드백 기록(SQLite Queue)'을 모아서, 전체 DB를 재-인덱싱할 때 1순위로 참고하는 심층 재학습 기능 구현.
    *   **얼굴 학습:** 사용자가 "이 사람은 성욱이야"라고 정정해둔 피드백 사진의 얼굴을 `InsightFace`가 집중 재학습하여 얼굴 인식 가중치 강화.
    *   **시공간 전파 학습:** 사용자가 수정한 '특정 날짜/장소' 피드백을 기반으로, 그 사진과 **유사한 시간대/유사한 배경 벡터(CLIP)** 를 가진 주변의 다른 사진들도 모조리 찾아내어 동일한 장소/시간으로 자동 일괄 적용(Propagate) 시켜주는 강력한 추론 군집화 알고리즘 도입 예정.
