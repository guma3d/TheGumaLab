# MyStockInsight 구현 로드맵 및 상태
이 문서는 사용자의 관심 주식 모니터링 및 AI 기반 맞춤형 주식 추천 기능을 제공하는 개인용 주식 대시보드 웹서비스를 구축하기 위한 작업 계획입니다. 현재는 **초기 기획 및 인프라 설계 단계**입니다.

## 🎯 주요 목표 (진행 중)
실시간 주가 데이터와 뉴스 API를 통합하고, Gemini API 기반의 AI 엔진을 활용하여 개인화된 종목 추천 및 트렌드 분석을 제공하는 직관적이고 세련된 웹 대시보드를 구축합니다.

---

## 📅 단계별 구현 및 완료 내역

### 1단계: 인프라 및 API 환경 세팅 (Docker 환경 구축) - 진행 예정
- [ ] `Dockerfile` 작성: 웹 백엔드 및 AI 분석 구동을 위한 기본 환경 구성 (모든 Python 코드는 명시적으로 `encoding='utf-8'` 적용하여 환경 문제 방지)
- [ ] `docker-compose.yml` 설정: 백엔드, 데이터베이스(관계형 데이터 관리용 SQLite & 임베딩용 ChromaDB), 인메모리 캐시(Redis), 스케줄러(정기 데이터 갱신) 컨테이너 구성
- [ ] 금융/뉴스 API 연동 준비: Yahoo Finance (또는 FinanceDataReader), News API 등의 키 확보 및 호출 테스트 진행
- [ ] Antigravity를 활용한 초기 프로젝트 보일러플레이트 코드 생성

### 2단계: 핵심 데이터 수집 및 백엔드 파이프라인 구축 - 진행 예정
- [ ] **관심 주식 현재가 로직**: 등록된 관심 주식의 현재 주가, 등락률 데이터를 주기적으로 Fetching 하는 워커 스레드 구현
- [ ] **API 캐싱 계층 구현**: 외부 API Rate Limit 초과 방지를 위해 Redis를 사용하여 일정 주기마다 호출된 데이터를 캐싱 및 프론트엔드에 제공
- [ ] **연관 뉴스 크롤링 및 요약**: 관심 주식과 관련된 주요 뉴스를 수집하고, Gemini를 거쳐 핵심 내용만 간결하게 요약해주는 파이프라인 구성
- [ ] DB 스키마 설계: 유저 포트폴리오, 관심 종목, 스크랩된 뉴스 기록을 효율적으로 관리할 SQLite 테이블 및 ChromaDB 벡터 저장소 구조 마련

### 3단계: AI 기반 주식 추천 엔진 도입 (Gemini API) - 진행 예정
- [ ] **맞춤형 주식 추천**: 기존 관심 주식의 섹터, 재무 상태, 뉴스 동향을 ChromaDB에서 검색 후 Gemini API를 활용하여 성향에 맞는 유망 종목 추천 로직 구현
- [ ] **떠오르는 주식 추천**: 시장 전체의 거래량 급증, 뉴스 언급량 증가 등을 바탕으로 현재 핫한 테마주 및 트렌드 주식 추천 기능 추가
- [ ] 프롬프트 최적화: 수집된 주식 데이터와 뉴스를 조합해 의미 있는 투자 인사이트를 도출하도록 Gemini 모델 프롬프트 튜닝

### 4단계: 프론트엔드 대시보드 연동 - 진행 예정
- [ ] 대시보드 UI/UX 설계: 현재 주가, 주요 뉴스, 추천 종목을 한눈에 볼 수 있는 직관적인 위젯/카드형 UI 개발
- [ ] **디자인 일관성 유지**: 전체적인 테마, 컬러 팔레트, 글꼴(Font) 및 컴포넌트 레이아웃 등은 기존 프로젝트(`YoutubeToDoc`, `ServerStatus`, 홈 포털)와 완벽한 시각적 일관성(Guma Lab 패밀리룩)을 갖도록 디자인
- [ ] 실시간 데이터 갱신: 웹소켓(WebSocket) 또는 적절한 주기의 폴링(Polling)을 활용하여 캐싱된 주가 및 뉴스 변동을 UI에 반영
- [ ] 반응형 웹 적용: 서피스 등 메인 작업 환경의 해상도에서도 쾌적하게 모니터링할 수 있도록 레이아웃 최적화

### 5단계: 배포 및 빌드 워크플로우 (Git -> HomeServer -> Docker)
개발과 배포는 랩탑과 홈서버가 일관되게 동기화되도록 다음 프로세스를 따릅니다:
1. **로컬 작업 및 푸시**: 랩탑(Gram)에서 개발 후 `/sync_github` 워크플로우로 로컬 변경사항을 깃허브 `main` 브랜치에 푸시 (`git add .`, `git commit`, `git push`)
2. **홈서버 최신화**: 홈서버에 SSH로 접속하여 깃허브 내용 동기화 (`git pull`)
3. **도커 컨테이너 재시작**: 홈서버에서 변경된 도커 환경을 재빌드하고 백그라운드 재시작 (`docker-compose down`, `docker-compose up -d --build`)
- **[요건 검증]**: 타겟 윈도우 환경 도커 컨테이너 가동 확인, 한글 깨짐 현상 체크 무결성, 홈서버 IP 및 단일 포트 로컬망 접속 확인
